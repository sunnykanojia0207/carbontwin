# CarbonTwin

> Understand your footprint. Decide what to change.

AI-powered personal carbon footprint platform that helps individuals understand, track, predict, and reduce their emissions using AI-powered insights.

## Features

- **Invisible Carbon Detector** — Snap a room photo; AI identifies appliances and estimates their carbon impact
- **Digital Climate Twin** — A personal carbon persona that evolves with your lifestyle
- **What-If Simulator** — Model the impact of solar panels, EVs, remote work, and more before committing
- **AI Carbon Negotiator** — A conversational advisor that finds reductions you'll actually keep
- **Carbon Forecasting** — 1/3/5-year projections with Paris 1.5°C alignment
- **Sustainability Goals** — Track commitments with milestones, achievements, and progress charts
- **Dashboard** — Real-time KPIs, trends, category breakdowns, and AI insights
- **Settings** — Full profile, preferences, theme, notifications, privacy, and API status management

## 🚀 Quick Start

### Prerequisites

- Node.js 18+ or Bun 1.0+
- A PostgreSQL database (or use the included SQLite for dev)

### Installation

```bash
# Clone the repository
git clone https://github.com/your-org/carbontwin.git
cd carbontwin

# Install dependencies
bun install

# Copy environment variables
cp .env.example .env
# Edit .env and fill in your values

# Set up the database
bun run db:push

# Start the dev server
bun run dev
```

Visit `http://localhost:3000` to see the app.

### Environment Variables

See [`.env.example`](.env.example) for the complete list. The critical ones:

| Variable | Required | Description |
|----------|----------|-------------|
| `DATABASE_URL` | ✅ | PostgreSQL connection string (or SQLite file path for dev) |
| `AUTH_SECRET` | ✅ | NextAuth secret (`openssl rand -base64 32`) |
| `GEMINI_API_KEY` | ❌ | Gemini API key (AI features degrade gracefully without it) |
| `GOOGLE_CLIENT_ID` | ❌ | Google OAuth client ID (enables Google login) |
| `GOOGLE_CLIENT_SECRET` | ❌ | Google OAuth client secret |

## 🗄 Database

The schema uses Prisma with 15 models:

- **User** + NextAuth infrastructure (Account, Session, VerificationToken)
- **Scan** + **Detection** — AI upload results
- **Appliance** — detected home devices
- **CarbonResult** — computed footprint aggregates
- **ClimateTwin** — user's carbon persona (1:1 with User)
- **Simulation** — what-if scenarios
- **Goal** + **GoalProgress** — reduction commitments
- **Recommendation** — AI-suggested actions
- **AIConversation** + **AIMessage** — chat history
- **Settings** — per-user preferences (1:1 with User)

```bash
# Push schema to database
bun run db:push

# Generate Prisma client
bun run db:generate

# Run migrations (production)
bunx prisma migrate deploy
```

## 🤖 AI Architecture

All AI features go through a unified facade (`src/lib/ai/index.ts`) that provides:

- **Rate limiting** — per-user, per-function token bucket
- **Caching** — TTL cache for stable responses (insights, suggestions)
- **Error handling** — typed `AiResult<T>` with standardized error codes
- **Fallback responses** — deterministic fallbacks so the UI never breaks
- **Secure env** — API keys validated at startup, never exposed to the client

## Code Quality

### Quality Gates

Every change must pass:

1. **TypeScript** — `strict: true`, zero `any` annotations, zero `@ts-*` suppression directives
2. **ESLint** — `npm run lint` returns zero errors, zero warnings
3. **Build** — `npm run build` compiles successfully
4. **Tests** — `npm run test` passes all 76+ tests

### Key Practices

- **No `any`** — TypeScript strict mode, explicit types everywhere
- **No emoji in code** — All visual indicators use `lucide-react` icons
- **Error boundaries** — `ErrorBoundary` component catches render crashes
- **Safe fetch** — `safeFetch<T>()` typed wrapper with retries & timeouts
- **Structured errors** — `ApiError` class with code, status, fieldErrors
- **User-safe messages** — `getUserFacingMessage()` never exposes internals
- **Unused import elimination** — ESLint `no-unused-vars` enforced
- **Console discipline** — `console.log` flagged, only intentional logging allowed
